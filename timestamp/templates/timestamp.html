Timestamp Microservice (FCC link)

User stories:
  1. I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: December 15, 2015)
  2. If it does, it returns both the Unix timestamp and the natural language form of that date.
  3. If it does not contain a date or Unix timestamp, it returns null for those properties.
Examples:
  • https://{{ site }}/api/timestamp/December%2015,%202015
  • https://{{ site }}/api/timestamp/1450137600 
{ "unix": 1450137600, "natural": "December 15, 2015" }
Try:

Enter unix timestamp, human readable date, or an invalid query
